## 0. 한 줄 요약 (One-liner)

> **"실패에서 배우고, 기술로 극복하고, 별에 닿아라."**
>
> 플레이어는 작은 우주 스타트업의 운영자가 되어, 연이은 발사 실패 속에서 기술을 업그레이드하며 성공 확률을 끌어올리고, 끝내 관측 가능한 우주의 끝까지 로켓을 보내는 **점진적 진행(Incremental) 시뮬레이터**입니다.

## 2. 기획 의도 (Creative Intent)

### 2.1 왜 지금 우주인가
- **2020년대 글로벌 최대 관심사 중 하나**: SpaceX, NASA Artemis, ESA의 유인/무인 우주개발 뉴스가 매주 쏟아지는 시대.
- **ESA(유럽우주국) 2024년 조사**: 유럽인 86%가 "우주 탐사는 청소년 세대에게 과학적 영감을 준다"고 응답.
- **STEM 교육 트렌드와 성인 우주 팬덤의 결합**: KSP·DSP·Mars Horizon이 입증한 **성인 우주 게임 시장**이 핵심 매출축. 청소년 STEM 메시지는 부수 마케팅 자산.

### 2.2 왜 이 장르인가
- **검증된 공식**: "증분(Incremental) / 업그레이드 루프"는 모바일(Egg Inc 1,000만 다운로드, 월 $30만 매출)과 스팀(Melvor Idle 누적 매출 $670만, Dyson Sphere Program 170만+ 장)에서 **공식화된 흥행 장르**.
- **극한의 단순함 × 장기 깊이**: 버튼 하나로 시작해서 수백 시간 플레이 가능한 콘텐츠 볼륨까지 확장 가능한 구조.
- **우주 소재 자체의 검증**: Kerbal Space Program 누적 400만 장($40), Spaceflight Simulator 모바일 누적 3,900만 다운로드 등으로 **글로벌 우주 게임 시장의 견고함**이 입증되어 있음.

### 2.3 설계 철학
> **"강제하지 않는다. 선택하게 만든다. 고통은 자원이다."**

- 플레이어는 항상 **스스로 리스크 수준을 선택**한다 (어떤 목적지에 도전할지, 어떤 업그레이드에 XP를 투자할지).
- **실패는 손실이 아니라 자원**이다 — 실패한 단계에서도 부분 XP를 획득하고, 누적 실패는 도감/통계/이야기를 만들어낸다.
- **돈을 써야만 재미있는 게임이 아니다** — 유료 아이템은 모두 "편의성·코스메틱·가속·투명한 확률 보정"이며, P2W는 금지.

## 3. 핵심 재미 포인트 (Fun Factor)

### 3.1 🎰 **재도전의 쾌감 — "한 번만 더"의 심리**
- 각 발사는 **N개 단계의 독립 확률 판정**(1~10단계). 모든 단계를 통과해야 목적지 완료.
- 9단계까지 성공하고 마지막에서 실패하는 **"아쉬움 루프"**가 슬롯머신/잭팟 심리를 자극.
- 0.8초 내외의 즉각적인 재발사 가능 → **끊김 없는 도파민 루프**.

### 3.2 📈 **성장의 실감 — "내가 강해진다"**
- **3축 성장 구조**:
  - **XP (세션)**: 지금 이 목적지에서만 쓰는 단기 성장
  - **Credit (영구)**: 발사장 전체를 업그레이드하는 장기 성장
  - **TechLevel (해금)**: 다음 목적지 문을 여는 스토리 축
- 업그레이드 한 개의 효과가 다음 발사에서 **즉각 체감**되는 빠른 피드백 루프.
- **구간형 확률 시스템**: 한 번 정복한 구간은 이후 발사에서 **자동으로 최고 확률** 적용 → "뒤로 가는 느낌" 제거, "앞으로 나아간다"는 실감만 남김.

### 3.3 🌌 **우주 스케일의 몰입 — "진짜 여행하는 느낌"**
- **11개 Zone, 100+ 개 목적지**: 대류권 → 카르만 선 → 달 → 화성 → 소행성대 → 목성 대적점 → 토성 고리 → 오르트 구름 → 프록시마 b → 오리온 성운 → **궁수자리 A\*(은하 중심 블랙홀)**까지.
- **2D 환경 점진 변화 연출**: Tier별 `ParallaxBackground` 텍스처 + `CanvasModulate` 색조 보간으로 하늘 색·별밀도가 자연스럽게 전환되는 **Sky Transition** 시스템.
- **마일스톤 사전 렌더 영상**: 첫 카르만 선 돌파 / 첫 달 도달 / 첫 화성 도달 / 첫 토성 고리 / 마지막 블랙홀 — 5~12초의 영상 컷으로 서사적 마침표. 모든 발사에 적용하지 않고 핵심 순간만 사용 → 도파민 루프 유지.
- 실제 천체 데이터 기반 (ISS, JWST, 아폴로 11호 착륙지 고요의 바다, 베누 소행성 등) → **학습적 만족감** + STEM 승인.

### 3.4 🔥 **전략의 긴장 — "도전할 것인가, 안전할 것인가"**
- **Stress / Overload / Abort 시스템** (T3 화성 이상부터 활성화):
  - 발사 실패 시 스트레스 게이지 누적 → 100% 초과 시 **시스템 과부하** 상태 진입.
  - 과부하 상태에서 발사 시 40~60% 확률로 **Abort 발동 + Credit 차감(수리비)**.
  - 플레이어는 "무리해서 더 갈 것인가, 5초 쉬어서 감쇠시킬 것인가"를 매 순간 판단.
- **단순한 클리커에 전략 레이어를 얹어** 장기 이탈을 방지하고, 숙련자에게 숙고의 재미를 제공.

### 4.4 벤치마크 게임의 유저 특성 (Why They Pay, How They Play)

심사위원의 가장 날카로운 질문은 *"그래서 그 유저들이 우리에게 올까?"* 입니다. 각 벤치마크의 유저 프로파일을 통해, **우리 게임이 어떤 유저층을 어떻게 흡수하는지** 명시적으로 매핑합니다.

#### 📱 [모바일] Spaceflight Simulator — "엔지니어/과학 호기심 청소년"
- **연령층**: 12~25세 중심, 남성 비중 80%+ 추정
- **플레이 스타일**: 매 세션 30~60분의 **"창작형" 플레이** — 로켓을 직접 설계·발사·실패 분석
- **결제 동기**: PC판 구매($5), 광고 제거, 부품 언락
- **🎯 우리 게임의 흡수 전략**: SFS의 "직접 만든다" 욕구는 우리에겐 부족하지만, 대신 **"100개 목적지 컬렉션"과 "도감"으로 같은 성취 욕구를 채움**. SFS는 진입 장벽이 높아 이탈하는 유저가 많은데, **우리의 캐주얼 접근성으로 그 이탈층을 받음**.

#### 📱 [모바일] Egg, Inc. — "성숙한 Idle 마니아"
- **연령층**: 25~45세 중심 (성인 시장)
- **플레이 스타일**: 하루 5~10분 짧은 세션을 **여러 차례 반복**, 오프라인 진행 중심
- **결제 동기**: Golden Eggs(영구 부스터) 구매, 시즌 보상, **계약(Contract) 시즌 패스**
- **수익 통찰**: 2025년 iOS 단독 월 $30만 — **출시 9년차임에도** 안정적 수익. "Idle은 단명 장르"라는 통념을 부순 사례
- **🎯 우리 게임의 흡수 전략**: Egg Inc의 BM(영구 부스터 + 시즌 콘텐츠 + 오프라인 진행)을 **거의 그대로 차용**. 차이점은 우리가 **시각적으로 훨씬 임팩트 있는 발사 연출**(Sky Transition + 마일스톤 영상)을 가진다는 것. Egg Inc는 닭 그래픽이 한계인 반면, 우리는 **우주 스케일을 시각화**.
- **참고**: Egg Inc의 협동(Co-op Contract)은 **싱글 오프라인 정책상 도입하지 않음**. 대신 일일 미션 / 시즌 챌린지로 같은 리텐션 효과 노림.

#### 📱 [모바일] Cell to Singularity — "교육 호기심 + 학부모 승인층"
- **연령층**: 10~17세(학생) + 25~45세(성인 호기심층) 양극화
- **플레이 스타일**: **느린 페이스 수용**, 게임 내 백과사전 읽으며 진행
- **결제 동기**: 진행 가속(Darwinium $0.99~$49.99), 광고 제거, 펫 코스메틱
- **포지셔닝 통찰**: "교육적 Idle"이라는 좁은 카테고리를 만들고 그 안의 1위로 자리잡음 — **카테고리 점유 전략의 모범**
- **🎯 우리 게임의 흡수 전략**: Cell to Singularity는 진화론, 우리는 천문학·우주공학. **교육 카테고리에서 직접 경쟁 없이 우주 분야 점유**.

#### 🖥️ [스팀] Kerbal Space Program — "엔지니어 마인드 하드코어 PC 게이머"
- **연령층**: 18~40세 (성인 PC 게이머)
- **플레이 스타일**: **세션당 3~8시간 장기 플레이**, 학습 곡선 가파름
- **포지셔닝 통찰**: 진입 장벽이 너무 높아 **"KSP 해보고 싶었지만 포기한 유저"가 수백만 명** — 이들은 이탈 후 갈 곳이 없음
- **🎯 우리 게임의 흡수 전략**: KSP의 **"포기자 시장"이 우리의 1차 타겟**. KSP가 보여주는 우주공학 깊이의 **10%만 추출**하고, 대신 **세션 길이 5~15분, 학습 곡선 거의 0**으로 만들어 흡수. 가격도 $14.99로 KSP의 1/3 미만.

## 5. 핵심 게임플레이 루프 (Core Gameplay Loop)

### 5.1 매크로 루프 (5~15분 세션)
```
[메인 화면 진입]
  → 발사 패널 / LAUNCH 버튼 즉시 노출
  → [LAUNCH 탭]
  → N단계 확률 판정 (각 단계 약 2초, 총 6~20초)
      ├─ 단계 성공: XP 지급 + 로켓 상승 연출 (2D AnimationPlayer)
      └─ 단계 실패: 자유낙하 연출 + 다음 발사 준비
  → 모든 단계 통과 시:
      ├─ 목적지 완료 (Credit + TechLevel 보상)
      ├─ Region 첫도달 뱃지 / 도감 갱신
      ├─ TechLevel 충족 시 다음 목적지 자동 해금
      └─ (조건 부합 시) 사전 렌더 영상 1회 재생
  → Upgrade 패널로
      ├─ Launch Tech (XP 소비, 세션형 5종)
      └─ Facility Upgrades (Credit 소비, 영구형 5종)
  → [메인 화면 → LAUNCH 반복]
```

### 5.2 마이크로 루프 (단일 발사, 6~20초)
```
LAUNCH 버튼
  → [LaunchService — 단계별 확률 판정]
  → Stage 1: 확률 판정 → 2초 상승 연출
  → Stage 2: ...
  → Stage N: ...
      ├─ 전부 성공 → launch_completed 시그널 (보상 연출 + 로컬 토스트)
      └─ 중간 실패 → stage_failed 시그널 (자유낙하, 다음 발사 준비)
```

체감 쿨타임 약 0.8초 — 0.5초 연출 + 0.3초 결과 처리.

### 5.3 메타 루프 (여러 날)
```
일일 접속
  → 오프라인 진행 요약 모달 (자동 발사 N회 / Credit +X / XP +Y)
  → 일일 보상 수령 (5~25 Credit + 부스터)
  → 일일 미션 3개 수행 (TechLevel 획득)
  → 주간 미션 TechLevel 캡(500) 소화
  → Program Level 상승 → 다음 Zone 해금
  → 도감/뱃지 컬렉션 완성
  → [다음 날 반복]
```

## 6. 주요 시스템 소개 (How the Fun Is Built)

> 각 시스템이 3장의 "재미 포인트"를 어떻게 실제로 구현하는지를 매핑합니다.

### 6.1 🎰 Launch Core — 다단계 확률 엔진 (재미 ①)
- **구간형 확률 구조**: 5개 Tier × 구간별 `baseChance`/`maxChance`. 정의는 `LaunchBalanceConfig.tres` (Godot `Resource`) 분리.

| Tier | 구간 | 스테이지 | baseChance | maxChance |
|---|---|---|---:|---:|
| 1 | Atmosphere (대기권) | 3~4 | 50% | 85% |
| 2 | Cislunar (달 궤도) | 5~6 | 44% | 78% |
| 3 | Mars Transfer | 7~8 | 36% | 72% |
| 4 | Outer Solar | 9 | 28% | 66% |
| 5 | Interstellar | 10 | 22% | 60% |

- **정복한 구간은 자동 상한**: 한 번 Tier 3을 깬 플레이어는 이후 발사에서 Tier 1~2 스테이지가 즉시 `maxChance`로 적용 → **체감 진행감 유지**.
- **스테이지 시간 2초** × N단계 → 발사 한 번이 6~20초의 짧은 "미니 드라마".

### 6.2 📈 Progression — 100개 목적지 × 11개 Zone (재미 ③)
- `D_01 ~ D_100` 선형 경로, 각 목적지는 `requiredStages / rewardCredit / rewardTechLevel / requiredTechLevel` 보유.
- **자동 진행 + 반복 도전 공존**: TechLevel 충족 시 자동으로 다음 목적지, 부족하면 현재 목적지를 반복해 TechLevel 파밍 가능.
- **Region / Region Mastery**: 같은 지역 목적지를 모두 완료하면 "Mars Pathfinder" 같은 마스터리 칭호.

### 6.3 📈 Economy — 3화폐 경제 (재미 ②)
| 화폐 | 역할 | 획득 | 소비 |
|---|---|---|---|
| **XP** | 세션형 단기 성장 | 스테이지 성공 시 | Launch Tech 5종 |
| **Credit** | 영구 성장 + 리스크 정산 | 목적지 완료 시 | Facility Upgrades 5종, Stress 수리비 |
| **TechLevel** (Program Lv.) | 해금 축 | 목적지 + 미션 | (직접 소비 없음, 단조 증가) |

- **역할 분리 원칙**: XP → Credit 교환 금지, Credit → TechLevel 교환 금지. 각 축의 압력이 섞이지 않게 설계.
- **세션 리셋**: 목적지 변경 시 XP와 세션 업그레이드는 초기화 → "이 목적지에 얼마나 투자할지" 선택이 전략이 됨.

### 6.4 🔥 Stress / Overload / Abort — 리스크 레이어 (재미 ④)
| Tier | 실패당 스트레스 | Abort 확률 | 수리비 |
|---|---:|---:|---:|
| 1~2 | 0 | 0% | 0 C |
| **3** | 10 | 40% | 300 C |
| **4** | 15 | 50% | 700 C |
| **5** | 20 | 60% | 1,500 C |

- **5초 이상 대기 시 초당 자연 감쇠** → AFK로 해소 가능 (강제 대기의 매너리즘 방지).
- **Launch Tech의 Stress Bypass 업그레이드**로 누적량 감소 가능 → 숙련자에게 "어느 축에 투자할지" 분기.

### 6.5 🌌 Cinematic & Sky Transition — 연출 시스템 (재미 ③, Godot 2D)
- **Sky/Lighting Transition**: Tier별 `SkyProfile.tres`에 등록된 배경 텍스처·색조·파티클 프리셋이 발사 지속시간/목적지 변경에 맞춰 점진 보간 (`Tween` + `CanvasModulate`).
- **`ParallaxBackground` 레이어드 배경**: 전경/중경/원경/별 3~5장. 발사 시 위로 스크롤되며 상승감 연출.
- **`Camera2D` 쉐이크**: 발사 / 실패 / 폭발 시 진폭 펄스.
- **Launch VFX**: `GPUParticles2D` — 불꽃 트레일 / 단계 분리 / 폭발 / 대기 돌파 섬광.
- **사전 렌더 영상 (`VideoStreamPlayer`)**: 마일스톤 5종 (10/25/50/75/100) + 핵심 첫도달. 5~12초, 720p, 스킵 가능.

### 6.6 📚 Meta / Collection — 장기 동기 부여
| 시스템 | 엔트리 수 | 설계 의도 |
|---|---|---|
| **Discovery / Codex** | 12 천체계 (Lite B) | 천체/천체계/영역 단위 수집 — "이 별에 대해 나는 얼마나 알고 있는가" |
| **Badge** | 19종 (Win 카운트 5 + 첫도달 14) | Steam Achievements / Google Play Games Achievements 매핑 |
| **Mission** | 주간 / 일일 | 주간 TechLevel 캡 500, 일일 3개 랜덤 풀 |
| **Best Records** | TotalWins / TechLevel / Best Tier | 로컬 베스트 + (V2) Steam Leaderboards / GPG Leaderboards 선택적 연동 |

### 6.7 🛠️ Auto Launch + Offline Progress — 편의성 시스템
- **Auto Launch**: T1 첫 클리어 또는 누적 10회 발사 시 무료 해금. IAP `Auto Launch Pass`로 속도 상한 +0.35 launches/s.
- **Offline Progress**: 종료 시 `last_saved_unix` 기록 → 재개 시 시간 차로 자동 발사 시뮬 (캡 8h, 자동 발사 미해금 시 0~1h).
- **오프라인 요약 모달**: 진입 시 "오프라인 동안 자동 발사 N회 / Credit +X / XP +Y" 표시.
- **메타 보너스**: 일일 로그인 보너스 / 시즌 컬렉션 보너스 / 도감 완성 보너스로 일일 복귀를 유도.

## 7. 수익 모델 (Monetization Strategy) — 모바일 + Steam 듀얼 트랙

### 7.0 왜 플랫폼별로 분리하는가

같은 게임이라도 **플랫폼마다 유저의 결제 심리·연령 구성·플랫폼 수수료·규제 환경이 전혀 다르기 때문에**, 단일 BM을 그대로 이식하면 어느 한 플랫폼에서는 반드시 실패합니다. 본 기획서는 **모바일(Android+iOS) / Steam** 두 플랫폼을 각각의 네이티브 유저 문화에 맞춰 별도 설계합니다.

| 축 | 모바일 (iOS / Android) | 스팀 |
|---|---|---|
| **플랫폼 수수료** | Apple/Google 30% (소규모 개발자 15%) | Steam 30% (매출 $10M 이후 25%) |
| **주 연령** | 전 연령, 중심은 20~40대 | 15~40대 PC 게이머 |
| **결제 방식** | IAP 강제 (외부 결제 차단) | Steam Wallet / 신용카드 직접 결제 |
| **주력 BM** | 소액 IAP × 다회 결제 + 광고 + 구독 | 프리미엄 1회 구매 + DLC |
| **규제** | IDFA/ATT, COPPA (13세 미만), 광고 동의 | GDPR, 환불 14일/2시간 정책 |
| **유저 이탈 주기** | 짧음 (첫 3일 70% 이탈) | 길음 (환불 기간 2주) |

> **핵심 통찰**: 모바일은 "많은 사람에게 조금씩", 스팀은 "적은 사람에게 한 번에 크게". 이 2가지 철학을 동일 게임에 공존시키는 것이 본 수익화 전략의 뼈대입니다.

### 7.1 공통 원칙 (양 플랫폼 모두 적용)
1. **P2W 금지**: 유료 아이템은 XP 배율·쿨다운 단축·확률 +소량 보정·리스크 정산 면제 등 **편의성·가속형**.
2. **확률 투명 공개**: 유료 확률 보정 상품은 "기본 5% → 8%" 식으로 수치 명시 (국내 확률형 아이템 표시 의무법 및 Apple/Google 가이드라인 준수).
3. **진행도 게이트**: 첫 5분 BM 모달 금지, Tier 해금에 맞춰 상품 점진 노출.
4. **아동 보호**: 13세 미만 유저에게는 광고·결제 CTA 완전 차단 (COPPA / GDPR-K 삼중 대응).

### 7.2 📱 **모바일 (iOS / Android) — F2P + 소액 IAP + 광고 + 구독**

  - Apple/Google의 **IAP(In-App Purchase) 외 결제 금지** (디지털 재화).
  - **IDFA/ATT** 이후 광고 어트리뷰션 정확도 하락 → 자체 리텐션 데이터 기반 최적화 필수.

## 8. 기술 스택 및 개발 역량

| 영역 | 선택 |
|---|---|
| **엔진** | **Godot 4.6** (GDScript / Standard) |
| **렌더러** | GL Compatibility (모바일 호환) / Windows 빌드는 Direct3D 12 |
| **아키텍처** | Godot **Autoload 싱글톤** 패턴 (GameState / EventBus / SaveSystem) + 서비스 노드 |
| **이벤트** | Godot Signal + EventBus 오토로드 |
| **데이터 영속** | 로컬 `user://savegame.json` (JSON, 스키마 버전 + 마이그레이션 훅) |
| **클라우드 세이브 (V2)** | Steam Cloud / Google Play Games / iCloud |
| **데이터 주도 설계** | 모든 정적 밸런싱 데이터를 `Resource` (`.tres`) 분리 |
| **UI** | Godot `Control` 노드 + `CanvasLayer` |
| **연출** | 2D `AnimationPlayer` + `Tween` + `ParallaxBackground` + `CanvasModulate` + `GPUParticles2D` + `VideoStreamPlayer`(사전 렌더 영상) |
| **결제** | Google Play Billing / Apple StoreKit / GodotSteam (`addons/godotsteam`) |
| **광고 (모바일)** | AdMob Godot plugin |

> **차별화 지점**: Godot 4.6의 **단일 코드베이스로 모바일/PC 동시 출시**가 가능한 점이 핵심 자산. Unity 대비 빌드/배포 복잡도가 낮고 라이선스 리스크가 없으며, 인디 우주 게임에 적합한 가벼운 풋프린트.

| **P7** | 13~14주차 | 밸런스 QA + 사운드/이펙트 타이밍 + 다국어(한/영) + 출시 제출 |
