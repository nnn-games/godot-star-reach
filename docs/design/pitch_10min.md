**[대본 — 45초]**
> 한 문장으로 설명드리면, 이 게임은 **실패가 쌓일수록 다음 성공이 커지는 우주 게임**입니다.
>
> SpaceX, NASA Artemis, 그리고 매주 쏟아지는 우주 뉴스 — **우주는 2020년대 글로벌 최대 관심사 중 하나**입니다. 유럽우주국이 2024년 조사한 결과, **유럽 청소년의 85%가 "우주 탐사가 자신에게 동기부여된다"**고 응답했습니다. 이게 저희가 노리는 시장의 크기입니다.
>
> 저희는 이 거대한 우주 관심을, **슬롯머신의 "한 번만 더" 심리와 결합**했습니다. 결과적으로 — **가장 짧은 5분 세션부터 가장 긴 수백 시간 장기 플레이까지** 모두 수용하는 단일 게임이 만들어졌습니다.
>
> 지금부터 이 게임이 왜 재미있고, 왜 돈이 벌리는지 6분과 3분에 나누어 말씀드리겠습니다.

**[전환 포인트]**
- "85%" 숫자를 또박또박 강조 — 청중 머릿속에 "큰 시장이 있구나"를 고정.
- "6분과 3분에 나누어" 부분에서 발표자가 손가락 2개 제스처 → 구조의 명확함 시각화.

**[중요 — 게임 레퍼런스 회피 원칙]**
> S2에서는 **특정 경쟁작 이름을 절대 언급하지 않는다**. 이유:
> - 심사위원은 모바일/PC 인디 게임을 잘 모름 → "그게 뭔데?" 의아함만 유발

> - 시장 검증을 위한 구체적 게임 데이터는 **S8(시장 검증)에서 일괄 제시**
> - S2의 역할은 **"우주는 큰 시장이다 + 우리는 새로운 심리 메커니즘이다"** 두 메시지 전달뿐

### 🎮 S3. 코어 루프 시연 (1:00 ~ 2:00)

**[비주얼]**
- 좌측 70%: 실제 인게임 발사 장면 30~40초 영상 (소리 있음 권장, 무음이면 자막).

  ```
    ↓
  [성공 → 상승] or [실패 → 낙하]

  ```

**[화면 구성]**
```
+------------------------+----------+
|                        | [메인]   |
|   [인게임 영상]        |   ↓      |
|   - 메인 화면          | [LAUNCH] |
|   - LAUNCH 버튼 탭     |   ↓      |
|   - 1단계 성공         | [확률    |
|   - 2단계 성공         |  판정]   |
|   - 3단계 성공...      |   ↓      |
|                        | [성공/   |
|                        |  실패]   |
|                        |   ↓      |
|                        | [반복]   |
+------------------------+----------+
```

**[대본 — 60초]**
> 게임의 핵심은 단순합니다. **LAUNCH 버튼을 탭**하고, 각 단계의 확률 판정을 **하나씩 통과**합니다.
>
> 보시는 것처럼 한 번의 발사가 **여러 개의 단계**로 나뉘어 있습니다. 대기권을 벗어나려면 3단계, 달에 가려면 5단계, 화성은 7단계, 성간 우주는 **10단계까지** 연속으로 성공해야 합니다.
>
> 각 단계의 성공률은 **사전에 정의된 확률 규칙에 따라 공정하게 판정**되고, 플레이어의 업그레이드 수준에 따라 점진적으로 올라갑니다.
>
> 여기서 가장 중요한 질문 하나가 나옵니다.
>
> "그럼... 10단계 중 **9단계까지 성공하고 마지막에서 실패하면** 어떻게 될까요?"

**[전환 포인트]**
- 마지막 질문을 던지고 0.5초 침묵 → **슬라이드 넘김**.
- 이 질문이 다음 슬라이드의 클라이맥스 장면으로 이어지는 **핵심 연결 고리**.

>

### 🌌 S5. 스케일 타임랩스 (3:00 ~ 4:00)

**[비주얼]**
- **풀스크린 타임랩스 영상 10초** (슬라이드 진입 즉시 자동 재생):
  - 지상 → 대기권 → 성층권 → 우주 → 달 → 화성 → 소행성대 → 목성 대적점 → 토성 고리 → 오르트 구름 → 프록시마 b → 오리온 성운 → **궁수자리 A\*** (은하 중심 블랙홀)
- 타임랩스 동안 우측 하단에 작은 카운터: `목적지 0 → 10 → 50 → 100`
- 영상 종료 후 정지 이미지: 은하 중심의 블랙홀 + 중앙 타이포 *"100+ 목적지. 실제 천체 데이터 기반."*

**[화면 구성]**
```

  0초: 파란 하늘 + 로켓
  2초: 구름 위 + 푸른 지구
  4초: 달 표면
  6초: 화성 + 붉은 사막
  8초: 목성 대적점
  10초: 블랙홀 사건의 지평선

  [정지 이미지]
  "100+ 목적지. 실제 천체 데이터 기반."
```

**[대본 — 60초]**
> 이 게임이 약속하는 것은 "진짜로 우주를 여행하는 느낌"입니다.
>
> 방금 보신 10초짜리 영상은, 실제 게임의 **Sky Transition 시스템**이 만들어내는 장면입니다. Tier별로 다른 배경 텍스처, 다른 색조, 다른 파티클 프로필이 등록되어 있고, 목적지 변경에 맞춰 **자연스럽게 블렌딩**됩니다.
>
> 목적지는 11개 Zone, **100개 이상**입니다. 국제우주정거장, 제임스 웹 우주망원경, 아폴로 11호 착륙지 "고요의 바다", 그리고 우리 은하의 중심 블랙홀 "궁수자리 A\*"까지.
>
> 이건 단순한 배경이 아닙니다. **NASA와 ESA의 실제 천체 데이터**를 기반으로 만들어졌기 때문에, 플레이어는 게임을 하면서 **실제 천문학 지식을 자연스럽게 접합니다**.
>
> 이게 KSP가 400만 장을 판매한 이유와 같습니다 — **우주 게임의 진정성**은 코어 팬을 만들고, 코어 팬은 장기 LTV를 만듭니다.

**[전환 포인트]**
- "진정성" 키워드는 베테랑 PC 게이머 심사위원에게 매우 강한 신호. 단순한 캐주얼이 아니라 **"지적 깊이가 있는 게임"**으로 인식시킴.

### 🗺️ S6. 100개 목적지 (4:00 ~ 5:00)

**[비주얼]**
- 전체: 수평으로 긴 **"우주 진행 맵"** 인포그래픽.
- 좌측부터 우측으로 11개 Zone이 아이콘으로:

## 3. Q&A 예상 대응 (심사위원 공격 시나리오)

### 3.1 "이미 비슷한 게임 많은데 왜 이게 살아남나?"
> 저희는 "발사 시뮬레이터" 시장이 아니라, **단계별 확률 + 100개 실제 천체 + Godot 듀얼 플랫폼 BM**의 조합으로 들어갑니다. 이 3가지를 동시에 갖춘 게임은 현재 없습니다.
>
> 또한 **각 벤치마크 게임의 한계가 곧 우리의 기회**입니다.
> - **Spaceflight Simulator**: 4천만 다운로드지만 진입 장벽이 높아 이탈자 다수 → 우리는 캐주얼 접근성으로 흡수
> - **Kerbal Space Program**: $40 하드코어, "해보고 싶었지만 포기"한 유저가 수백만 → 우리는 $14.99 캐주얼 Idle로 흡수
> - **Mars Horizon**: ESA 공식 협력의 좋은 게임이지만 "딱딱한 시뮬 인터페이스"로 niche에 갇힘 → 우리는 같은 컨셉을 Idle로 풀어 대중화
> - **Egg Inc**: 9년차에도 월 $30만 안정 매출이지만 시각적 한계(닭 그래픽) → 우리는 Sky Transition으로 우주 스케일 시각화

### 3.2 "왜 싱글 오프라인 구조인가?"
> **인디 규모에서 가장 안정적이고 ROI가 높은 구조가 싱글 오프라인 모바일/Steam 듀얼 트랙**입니다. 근거는:
> - 모바일/Steam은 플랫폼 수수료 30%로 매출 효율이 우수 (소규모 개발자 프로그램 적용 시 15%)
> - 인프라 비용이 0 (서버 / 동기화 / 치트 방지 시스템 불필요)
> - Godot 4.6 단일 코드베이스로 모바일+PC 동시 출시가 더 큰 시장 도달 가능
> - 오프라인 진행은 Idle / Incremental 장르의 핵심 매력

### 3.3 "10단계 확률이면 초반 이탈률 치명적이지 않나?"
> T1 대기권은 3단계이고 **기본 성공률 50%**입니다. 첫 2~3발은 튜토리얼 보정으로 **100% 성공** 보장합니다. 연속 실패 시 **Pity System**(연민 보정)으로 확률이 자동 보정됩니다. 10단계는 엔드게임 전용이고, 그 단계까지 도달한 유저는 **이미 수십 시간 투자한 하드코어층**이므로 이탈 리스크가 낮습니다.

### 3.4 "2플랫폼 동시 운영하면 개발비 폭증 아닌가?"
> 핵심 게임 로직(확률 판정, 진행 시스템, 경제, 도감)은 **Godot 단일 코드베이스로 100% 공유**됩니다. 플랫폼별로 분리되는 것은 **결제 API(Google Play Billing / StoreKit / GodotSteam), 광고 API(AdMob), 실적 API(Steam Achievements / Google Play Games / iOS Game Center)** 레이어뿐입니다.
>
> 싱글 오프라인이라 **서버 운영 비용이 0**입니다. 클라우드 세이브는 V2에서 Steam Cloud / Google Play Games Saved Games / iCloud의 플랫폼 표준 API를 사용합니다.

### 3.5 "P2W가 전혀 없으면 매출이 약하지 않나?"
> 확률 보정 상품(Guidance Module, Trajectory Surge)은 있지만 **+5%p 수준으로 제한**하고 수치를 공개합니다. "돈 안 써도 공평하다"는 인식이 **장기 리텐션과 코어 팬덤 형성**을 결정적으로 높이기 때문에, **단기 ARPU 포기 → 장기 LTV 확보** 트레이드오프를 의도적으로 선택했습니다.
>
> **증거**: **Melvor Idle**은 마이크로트랜잭션 0%로 스팀에서만 누적 매출 **$670만 (약 67억 원)**을 달성했습니다. P2W 없는 Idle BM이 충분히 작동한다는 직접적 증명입니다.

### 3.6 "세일즈 수치는? 얼마나 팔릴 거라 보나?"
> **벤치마크 기반 보수적 추정**:
> - **스팀**: KSP($40)가 400만 장, DSP($20)가 170만 장, Mars Horizon($20)이 20~50만 장. 저희($15)는 출시 첫해 **5만~15만 장** 목표
> - **모바일**: Spaceflight Simulator(무료)가 3,900만 다운로드, Egg Inc(무료)가 1,000만 다운로드. 저희는 출시 첫해 **누적 30~50만 다운로드, ARPDAU $0.05~$0.10** 목표
>
> 합산하면 출시 첫해 약 **10~20억 원 매출** 추정 (상세는 별도 수익화전략서).

### 3.7 "이미 시스템이 있다고 했는데 진짜인가?"
> `docs/systems/INDEX.md`에 **29개 시스템 카탈로그**가 정리되어 있고, 각 시스템의 책임 / Config / 데이터 / 의존성이 문서화되어 있습니다. Phase 1 (확률 엔진 + 세이브 시스템)은 이미 코드 레벨에서 시작되어 있습니다.

### 3.8 "벤치마크 게임 유저들이 정말 우리 게임으로 올까?"
> 우리는 각 벤치마크의 **유저 프로파일과 결제 동기를 정확히 매핑**해두었습니다.
> - **KSP의 진입 포기자** → 캐주얼 접근성으로 흡수 (수백만 명 규모)
> - **Egg Inc 유저** → 시각적 임팩트와 우주 테마로 차별화 (월 $30만 매출 시장)
> - **Cell to Singularity 유저** → 같은 "교육적 Idle" 카테고리 내 우주 분야 점유
> - **Spaceflight Simulator 이탈자** → 100개 목적지 + 도감으로 같은 성취 욕구 충족
>
> **각 유저층의 5~10%만 흡수해도 누적 수십만 명 규모**입니다. 자세한 유저 특성 매핑은 `docs/design/game_overview.md` 4.4 섹션에 있습니다.

### 3.9 "솔로/소규모 인디는 지원이 부족할 텐데, 마케팅은 어떻게?"
> Gnorp Apologue가 좋은 사례입니다. 노르웨이 솔로 개발자가 **위시리스트 단 1,700개로 시작**해서 1년 만에 **10만 장**을 팔았는데, **거의 100% YouTube와 Steam 유기 추천**으로 달성했습니다.
>
> 저희 전략: 출시 전 **YouTube 인플루언서 대상 키 배포** (우주/Idle 카테고리), Steam Next Fest 데모 참여, 출시 첫 주 인디 게임 큐레이터 연락. **광고비 ZERO로 시작해 유기 확산을 기본 전략**으로 둡니다.
