# DestinationConfig — 목적지 콘텐츠 구조 기준 문서

> 문서 유형: 콘텐츠 구조 기준 문서
> 개정일: 2026-04-24
> 목적: 100개 목적지 콘텐츠 풀의 `목적지 / 도감 / 첫 도달 뱃지 / 반복 탐사 마스터리` 관계를 하나의 구조로 정리.
> 관련 문서: `docs/celestial_codex_design_plan.md`, `docs/celestial_codex_plan.md`, `docs/rocket_launch_implementation_spec.md`

---

## 1. 문서 역할

이 문서는 더 이상 "목적지 100개를 상세 설명과 함께 나열한 참고 문서"가 아니다.  
이제는 아래 4개 층을 동시에 정리하는 `콘텐츠 구조 기준 문서`다.

1. `Destination`
2. `ExplorationEntry`
3. `Region Arrival Badge`
4. `Region Mastery`

즉, 이 문서는 "어떤 목적지를 만들 것인가"만이 아니라 "그 목적지가 어떤 지역 묶음에 속하고, 도감과 보상 루프에서 어떤 역할을 하는가"까지 함께 정의한다.

## 1.1 현재 구현과의 관계

이 문서의 목적지 / 지역 / 마스터리 구조는 Godot 4.6 프로젝트에서 다음 리소스로 구현된다.

핵심 전제:

- `목적지 이름 / 지역 묶음 / 도감 매핑 / 뱃지 / 마스터리`는 이 문서를 기준으로 정리한다.
- Godot 리소스 파일 (`data/destination_config.tres`, `data/codex_config.tres`, `data/badge_config.tres`, `data/region_mastery_config.tres`)은 이 문서의 정의를 1:1로 반영한다.
- `T6` 성간 우주 콘텐츠는 현재 활성 범위가 아니라 `미래 확장 (DLC: Interstellar Frontier)`으로 내린다.

---

## 2. 확정된 기획 전제

현재까지 확정된 기준은 아래와 같다.

- `Destination`와 `ExplorationEntry`를 분리한다.
- 목적지 성공 시 해당 지역/천체 도감이 열린다.
- 도감 진행은 `같은 목적지 반복`이 아니라 `같은 지역의 서로 다른 목적지 성공`으로 쌓는다.
- `뱃지`는 지역 첫 도달 보상으로 사용한다.
- 반복 탐사 성취감은 `마스터리`로 설계한다.
- 마스터리 보상은 `칭호`, `프레임`, `패치`, `발사 연출 코스메틱` 같은 표현형 보상 위주로 간다.
- 도감 열람 자체는 강제하지 않는다.
- 현재 실제 출시 우선순위는 `T1~T5 + Codex Lite B + 첫 도달 뱃지`까지를 우선 범위로 본다.
- `지역 마스터리`는 구조는 지금 확정하되, 상세 보상 테이블은 1차 구현 직후 확장 가능한 단계로 둔다.

## 3. 콘텐츠 층 구조

| 층 | 단위 | 해금/진행 기준 | 플레이어 체감 | 보상 역할 |
|---|---|---|---|---|
| Gameplay | `Destination` | 발사 성공 1회 | "이 목표를 클리어했다" | Credit / TechLevel / 다음 목표 |
| Collection | `ExplorationEntry` | 관련 목적지 첫 성공 | "이 지역을 알게 됐다" | 도감 열람 / 섹션 해금 |
| Milestone | `Arrival Badge` | 해당 지역 첫 도달 | "여기까지 왔다" | 기념 배지 |
| Mastery | `Region Mastery` | 같은 지역의 서로 다른 목적지 반복 성공 | "이 지역을 깊게 탐사했다" | 칭호 / 프레임 / 패치 / 코스메틱 |
| Future Meta | `Set / Collection` | 복수 지역 완성 | "이 계열 전체를 정복했다" | 세트 보상 |

핵심은 `첫 도달의 성취`와 `반복 탐사의 성취`를 분리하는 것이다.

## 4. 현재 활성 범위 잠금

## 4.1 Gameplay 활성 범위

- 현재 구현 기준 활성 티어: `T1 ~ T5`
- 현재 코드 기준 최고 목적지 티어: `5`
- 현재 발사 연출 route도 `tier 1~5`만 기준으로 동작

따라서 아래 원칙으로 잠근다.

- `T6 성간 우주`는 현재 문서에서 `미래 확장 콘텐츠 풀`로 유지한다.
- 현재 실구현과 바로 연결되는 목적지 구조 문서는 `T1~T5`를 기준으로 설계한다.

## 4.2 Codex 1차 출시 범위

도감 1차 출시는 `Lite B` 구조로 간다.

- 핵심 엔트리 수: `12개`
- 엔트리당 섹션 수: `3~4개`
- 엔트리당 팩트 카드: `3개`
- 지역 첫 도달 뱃지: 포함
- 지역 마스터리: 구조 확정, 보상은 1차 간소화 또는 1.1차 확장

`지역 단위`는 아래 세 가지 시스템의 기준이 된다.

- 첫 도달 뱃지

## 7. 뱃지 구조

## 7.1 첫 도달 뱃지 원칙

뱃지는 `지역 첫 도달` 보상으로 사용한다.

규칙:

- 같은 지역에 속한 목적지를 처음 하나라도 성공하면 뱃지 지급
- 뱃지는 `Region` 기준으로 1회만 지급
- 뱃지는 `첫 도달의 기념`이지, 반복 탐사 보상이 아니다

좋은 예:

- 첫 달 관련 목적지 성공 -> `Lunar Pathfinder`
- 첫 목성 관련 목적지 성공 -> `Jovian Observer`

나쁜 예:

- 같은 지역 목적지를 5번 반복 성공할 때마다 뱃지 지급
- 쉬운 목적지를 반복해 뱃지를 여러 장 수급

## 7.2 뱃지와 도감의 관계

- 뱃지 = 해당 지역에 도달했다는 `기념`
- 도감 = 해당 지역을 `이해하고 수집`하는 축

즉, 둘은 연결되지만 같은 시스템이 아니다.

## 8. 반복 탐사 마스터리 구조

## 8.1 마스터리 목적

반복 탐사의 성취감을 뱃지 대신 `마스터리`로 설계한다.

플레이어가 느껴야 하는 감정은 아래와 같다.

- 첫 도달: "여기까지 왔다"
- 반복 탐사: "이 지역을 깊게 판다"
- 마스터리 완성: "이 지역 전문가가 됐다"

## 8.2 마스터리 기본 규칙

- 마스터리는 `같은 지역의 서로 다른 목적지 성공`으로 쌓는다.
- 같은 목적지 반복 성공은 기본 마스터리 포인트에 반영하지 않는다.
- 필요 시 `연속 탐사 체인`은 부가 보너스로만 쓴다.
- 마스터리 보상은 표현형 보상 위주로 설계한다.

## 8.3 탐사 난이도 등급 적용

각 지역에는 `탐사 난이도`를 부여한다.  
이 값은 발사 난이도를 다시 올리는 값이 아니라, `마스터리 완성 조건`의 무게를 조절하는 값이다.

| 탐사 난이도 | 의미 | 권장 마스터리 기준 |
|---|---|---|
| 1 | 초반 지역 | 서로 다른 목적지 2/4/6개 |
| 2 | 비교적 익숙한 심화 지역 | 서로 다른 목적지 2/4/7개 |
| 3 | 중반 주력 지역 | 서로 다른 목적지 3/5/8개 |
| 4 | 외태양계 고난도 지역 | 서로 다른 목적지 3/6/9개 + 고난도 목적지 1개 |
| 5 | 종반 / 외곽 / 확장 지역 | 서로 다른 목적지 4/7/10개 + 고난도 목적지 2개 |

여기서 `고난도 목적지`는 해당 지역 내 상위 `requiredStages` 밴드에 속한 목적지를 뜻한다.

## 8.4 마스터리 보상 레이어

| 단계 | 의미 | 권장 보상 |
|---|---|---|
| `M1 Surveyed` | 지역을 맛보기로 탐사 | 도감 스탬프 / 작은 UI 마커 |
| `M2 Explorer` | 지역 핵심 포인트 파악 | 프로필 패치 |
| `M3 Specialist` | 지역 주력 탐사자 | 칭호 |
| `M4 Veteran` | 고난도 탐사 포함 | 도감 프레임 / 명찰 장식 |
| `M5 Master` | 지역 심화 완성 | 발사 연출 코스메틱 / 기념비 장식 |

원칙:

- 마스터리 보상은 `표현형`
- 직접적인 파워 상승 없음
- 반복 지급 가능한 Credit/Fame 파밍 구조 금지

## 8.5 연속 탐사 체인

체인은 마스터리의 핵심이 아니라 `보너스 챌린지`다.

규칙 권장안:

- 같은 지역의 `서로 다른 목적지`를 연속 성공할 때만 체인 증가
- 실패 또는 다른 지역으로 이동하면 체인 종료
- 체인은 배지 대신 `임시 명예 기록` 또는 과제용으로 사용

## 9. 티어별 콘텐츠 역할 재정의

기존 문서는 티어를 곧바로 콘텐츠 주제로 보았다.  
확정안에서는 `티어`는 난이도/진입 구조이고, `지역`은 콘텐츠 의미 구조다.

| 티어 | 현재 역할 | 콘텐츠 의미 |
|---|---|---|
| `T1` | 온보딩 / 저궤도 익숙해지기 | `REGION_NEAR_EARTH` 소개 |
| `T2` | 첫 외계권 진입 | `REGION_MOON` + `SYSTEM_EARTH_MOON` |
| `T3` | 내태양계 분화 | 화성 / 금성 / 수성 / 소행성대로 분기 |
| `T4` | 외태양계 심화 | 목성 / 토성 / 얼음 거성 계열 분화 |
| `T5` | 태양계 외곽 / 경계권 | 카이퍼 벨트 / 헬리오스피어 프론티어 |
| `T6` | 미래 확장 | 성간 우주 전용 콘텐츠 |

중요한 변화:

- 티어는 더 이상 `도감 단위`가 아니다.
- 같은 티어 안에서도 여러 지역이 분리될 수 있다.
- 같은 지역이 여러 티어에 걸쳐 확장될 수도 있다.

## 10. 현재 콘텐츠 풀 재배치 방침

## 10.1 유지할 것

- 기존 100개 목적지 콘텐츠 풀
- 실제 천문 정보 기반의 교육 톤
- 티어 기반의 점진적 난이도 상승

## 10.2 바꿀 것

- `목적지 = 교육 콘텐츠 1개`라는 사고방식

## 12. 최종 요약

이제 목적지 콘텐츠 구조는 아래 한 줄로 이해하면 된다.

> `목적지`는 클리어할 목표이고, `지역`은 기념과 반복 탐사의 단위이며, `도감`은 지식 구조이고, `마스터리`는 반복 탐사의 성취 장치다.

따라서 이 문서의 역할은 단순 목적지 나열이 아니라, 이후 구현과 운영이 따라야 할 `콘텐츠 구조 기준`이다.
