# 🚀 발사 성공률 밸런스 설계

> **문서 유형**: 구현 반영 기획 문서
> **작성일**: 2026-04-24
> **목적**: Godot 4.6 클라이언트에 적용되는 발사 성공률 밸런스 구조를 기획 관점에서 정리
> **관련 코드**: `data/launch_balance_config.tres` (Godot Resource), `scripts/services/launch_service.gd`, `scripts/autoload/save_system.gd`, `scripts/services/destination_service.gd`

---

## 1. 설계 목표

현재 발사 밸런스는 다음 3가지를 동시에 만족하도록 설계한다.

1. 초반 구간은 쉽게 성공한다.
2. 뒤 구간으로 갈수록 같은 업그레이드 수준에서도 시작 성공률이 낮아진다.
3. 이미 한 번 돌파한 구간은 이후 상위 목적지 도전에서 높은 성공률로 재사용된다.

즉, 플레이어는

- 처음에는 `성공 경험`을 빠르게 얻고
- 상위 목적지에서는 `새 구간`에서 다시 긴장감을 느끼며
- 이전에 정복한 구간은 `숙련 구간`으로 안정적으로 통과하는 구조를 경험한다.

---

## 2. 현재 적용 구조

### 2.1 구간 개념

성공률은 더 이상 "모든 단계가 같은 확률"이 아니다.  
현재는 스테이지 번호를 5개의 구간으로 나누고, 구간마다 별도 기본 성공률을 둔다.

| 구간 티어 | 스테이지 범위 | 역할 | 기본 성공률 | 재도전 최고 성공률 |
|---|---|---|---|---|
| T1 | 1~4 | 초반 대기권 / 저궤도 구간 | 50% | 85% |
| T2 | 5~6 | 달권 진입 구간 | 44% | 78% |
| T3 | 7~8 | 화성권 진입 구간 | 36% | 72% |
| T4 | 9 | 외태양계 진입 구간 | 28% | 66% |
| T5 | 10 | 최종 성간 구간 | 22% | 60% |

### 2.2 해석

- `기본 성공률`: 해당 구간을 아직 한 번도 완주하지 못한 플레이어가 시작할 때의 기준값
- `재도전 최고 성공률`: 그 구간을 이미 돌파한 뒤, 상위 목적지에서 다시 만났을 때 적용되는 안정 성공률

예를 들어 T2 목적지를 처음 도전하는 플레이어는

- 스테이지 1~4는 이미 익숙한 T1 구간으로 높은 성공률
- 스테이지 5~6은 새로 열린 T2 구간으로 낮은 시작 성공률

을 갖는다.

## 3. 최종 계산식

### 3.1 업그레이드 보너스

현재 성공률 보정에 합산되는 업그레이드/IAP 보너스는 다음과 같다.

| 출처 | 최대치 | 설명 |
|---|---|---|
| Launch Tech `Engine Precision` | +40%p | 세션형 업그레이드 (XP 소비, 목적지 변경 시 리셋) |
| Facility `Engine Tech` | +10%p | 영구 업그레이드 (Credit 소비) |
| IAP `Guidance Module Pass` | +5%p | 영구 IAP ($5.99), 상시 적용 |
| IAP `Trajectory Surge` | +3%p | 소모 IAP ($1.99), 30분 시간제 버프 |

상시 적용 가능 보너스 합산 최대 `+55%p`. Trajectory Surge 활성 중에는 일시적으로 `+58%p`까지 도달.

### 3.2 계산 규칙

각 스테이지의 확률은 아래 규칙으로 계산한다.

```gdscript
var upgrade_bonus: float = (
    engine_precision_bonus
    + engine_tech_bonus
    + guidance_module_bonus
    + (trajectory_surge_bonus if SurgeService.is_active() else 0.0)
)

if segment_tier <= GameState.highest_completed_tier:
    stage_chance = segment.max_chance
else:
    stage_chance = min(segment.base_chance + upgrade_bonus, segment.max_chance)
```

### 3.3 핵심 의도

- 새 구간은 업그레이드를 타더라도 `구간 상한`을 넘지 못한다.
- 이미 깬 구간은 업그레이드 수치와 무관하게 즉시 `구간 최고 성공률`로 고정된다.
- 그래서 상위 목적지로 갈수록 "앞부분은 안정적, 마지막 미개척 구간만 어렵다"는 느낌이 만들어진다.

## 4. 이전 구간 재적용 규칙

플레이어 데이터에는 `highestCompletedTier`가 저장된다.

| 값 | 의미 |
|---|---|
| 0 | 아직 어떤 구간도 정복하지 못함 |
| 1 | T1 구간 완주 경험 있음 |
| 2 | T2 구간까지 완주 경험 있음 |
| 3 | T3 구간까지 완주 경험 있음 |
| 4 | T4 구간까지 완주 경험 있음 |
| 5 | T5 구간까지 완주 경험 있음 |

목적지 완료 시 해당 목적지의 `tier`로 이 값을 갱신한다.

### 4.1 예시

- 플레이어가 T1 목적지를 한 번이라도 클리어하면 이후 T2 이상 목적지에서 스테이지 1~4는 항상 `85%`
- 플레이어가 T2 목적지를 클리어하면 이후 T3 이상 목적지에서 스테이지 5~6은 항상 `78%`
- 플레이어가 T4까지 돌파한 상태에서 T5를 도전하면 스테이지 1~9는 모두 높은 안정 구간이 되고, 스테이지 10만 새로운 난관이 된다

---

## 5. 목적지 티어와 스테이지 수의 관계

현재 목적지 데이터는 다음 식으로 밸런스 구간과 연결된다.

| 목적지 티어 | 현재 목적지 요구 스테이지 | 실제로 새로 열리는 구간 |
|---|---|---|
| T1 | 3~4 | T1 |
| T2 | 5~6 | T2 |
| T3 | 7~8 | T3 |
| T4 | 9 | T4 |
| T5 | 10 | T5 |

즉 목적지 티어가 올라갈수록 "기존 구간 위에 새 구간이 1단계씩 추가되는 구조"다.

---

## 6. UI 표기 규칙

대기 상태에서 UI에 보이는 `현재 성공률`은 다음 의미를 가진다.

- 목적지 전체 평균값이 아니다.
- 첫 스테이지 확률도 아니다.
- **현재 목적지에서 아직 미완료인 첫 구간의 성공률**이다.

이렇게 표시하는 이유는 다음과 같다.

- 상위 목적지에서 초반 스테이지만 보고 `85%`로 표시하면 실제 체감 난이도보다 너무 쉬워 보일 수 있다.
- 플레이어가 지금 실제로 넘어야 하는 "새 구간"의 체감 난도를 보여주는 쪽이 더 직관적이다.

발사 중에는 각 스테이지 결과 패킷에서 해당 스테이지의 실제 확률을 그대로 표시한다.

## 7. 실제 플레이 예시

### 7.1 신규 플레이어, 업그레이드 없음, T1 목적지

- `highestCompletedTier = 0`
- `upgradeBonus = 0`
- 스테이지 1~4 확률 = `50%`

의도:
- 첫 구간에서 약 절반 수준으로 성공 경험을 제공

### 7.2 T1 클리어 후, 업그레이드 없음, T2 목적지

- `highestCompletedTier = 1`
- 스테이지 1~4 = `85%`
- 스테이지 5~6 = `44%`

의도:
- 초반은 빠르게 통과
- 새로 열린 T2 구간에서만 다시 실패와 투자 동기 발생

### 7.3 T2 클리어 후, 업그레이드 보너스 +18%p, T3 목적지

- `highestCompletedTier = 2`
- 스테이지 1~4 = `85%`
- 스테이지 5~6 = `78%`
- 스테이지 7~8 = `min(36% + 18%, 72%) = 54%`

의도:
- 이전 구간은 안정적
- 새 구간은 투자 효과가 분명히 보이지만 아직 긴장감 유지

### 7.4 풀 업그레이드 상태, T5 목적지

- `upgrade_bonus = 55%p` (Trajectory Surge 적용 시 일시적으로 58%p)
- T1 = `85%`
- T2 = `78%`
- T3 = `72%`
- T4 = `66%`
- T5 = `min(22% + 55%, 60%) = 60%` (Surge 중 `min(22% + 58%, 60%) = 60%`)

의도:
- 최종 구간도 완전히 쉬워지지 않도록 상한 유지
- 엔드게임 도전 가치 보존

## 8. 현재 밸런스의 장점

### 8.1 초반 이탈 완화

기존 고정 20% 구조와 달리 초반 T1은 50%에서 시작한다.  
그래서 첫 성공 경험까지 걸리는 시간이 크게 줄어든다.

### 8.2 상위 티어 체감 분리

구간별 기본값이 점점 낮아지므로

- T1은 "배우는 구간"
- T2~T3는 "투자 체감 구간"
- T4~T5는 "엔드게임 압축 구간"

으로 성격이 분리된다.

### 8.3 성장 체감 강화

이전 구간을 최고 성공률로 재사용하기 때문에 상위 티어에 올라갈수록 플레이어는

- "예전엔 어렵던 구간이 이제는 안정적이다"
- "지금 어려운 건 새 구간뿐이다"

라는 성장을 명확하게 체감한다.

---

## 9. 향후 조정 포인트

현재 수치는 첫 적용안이다. 실제 플레이 테스트에서 아래를 확인하며 조정한다.

1. T1 초반 성공률 50%가 너무 높아 학습 구간이 지나치게 짧지 않은지
2. T2 `44%`, T3 `36%`가 업그레이드 없을 때 과도한 막힘을 만들지 않는지
3. T5 `60%` 상한이 엔드게임 기준으로 너무 낮거나 높지 않은지
4. `Engine Precision`과 `Engine Tech` 투자 우선순위가 지나치게 고정되지 않는지
5. IAP 보너스가 특정 구간 상한을 너무 빨리 채워 밸런스를 무력화하지 않는지
