# Godot Agent-Driven Development (ADD) 작업 흐름 설계

이 문서는 사용자의 수동 프로젝트 조작을 최소화하고, 모든 주요 스크립팅과 설정을 AI(에이전트) 기반으로 자동화하기 위한 작업 파이프라인 설계안입니다. 

Godot 엔진은 씬(`.tscn`)과 설정(`.godot`) 파일이 모두 순수 텍스트(Text)로 이루어져 있어 AI가 직접 시스템의 프로젝트 파일을 조작하기에 매우 적합한 환경을 제공합니다.

## User Review Required

> [!IMPORTANT]
> 아래의 워크플로우 제안을 읽어보시고, 사용자가 직접 수동으로 하고 싶은 범위(예: "씬(Scene) 노드 배치는 내가 할게, 코드만 짜줘" 혹은 "씬 구성부터 스크립트 작성까지 전부 텍스트 파일로 AI가 알아서 해줘")를 결정해 주시면 이에 맞춰 향후 개발을 진행하겠습니다.

## 🚀 Agent-Driven 작업 흐름 (Workflow) 

### 1단계: 목표 지시 (User)
* 사용자는 만들고 싶은 기능, 기획, 또는 발생한 버그를 AI에게 자연어로 단순히 지시합니다.
* 예: *"스페이스바를 누르면 플레이어가 점프하는 기능을 만들어줘."*, *"현재 화면에 점수판 UI를 추가해줘."*

### 2단계: 구조 설계 및 권한 수행 (AI)
* **단독 분석**: AI가 지시사항을 분석하여 필요한 노드 계층 구조(Tree)와 GDScript 기능 구조를 설계합니다.
* **직접 파일 수정**: AI가 시스템 접근 권한(`write_to_file`, `replace_file_content` 등)을 이용해 백그라운드에서 직접 `.gd`(스크립트) 파일과 `.tscn`(씬 구조) 파일을 생성 및 수정합니다.
* **프로젝트 설정**: 필요시 입력 맵핑(Input Map)이나 자동 실행 로직(Autoload)도 AI가 `project.godot` 파일을 직접 수정하여 세팅합니다.

### 3단계: 확인 및 에디터 로드 (User & AI)
* AI가 파일 작성을 완료하면, 사용자에게 작업 내역의 요약(Walkthrough)을 제공하고 에디터에 변경 사항이 잘 갱신되었는지 확인을 요청합니다.
* 사용자는 Godot 에디터를 켜둔 채로 AI의 작업 완료 메시지가 뜨면, 에디터가 파일을 자동으로 다시 로드(Reload)하는 것만 확인합니다.

### 4단계: 테스트 및 피드백 (User)
* 사용자는 곧바로 **`F5`** 키를 눌러 게임을 실행(Test)합니다.
* 시각적인 결과, 조작감, 혹은 발생한 에러 메시지를 AI에게 그대로 전달합니다.
* 발견된 에러 로그는 복사해서 붙여넣어 주시면 AI가 즉시 원인을 파악해 코드를 자가 수정(Self-correction)합니다.

## 🛡️ AI 주도 개발을 위한 특수 전략

Godot의 씬(`.tscn`) 파일은 구조가 복잡해지면 AI가 텍스트만으로 직접 수정 시 충돌이 발생할 수 있습니다. 수동 조작을 최소화하기 위해 다음 규칙을 적용할 것을 제안합니다:

1. **코드 중심 생성 (Procedural Generation)**: 복잡한 UI나 동적 노드들은 씬 파일에 직접 우겨넣기보다, `_ready()` 함수 내에서 GDScript 코드를 통해 동적으로 생성(`Node.new()`)하고 배치하도록 코딩합니다. AI가 가장 잘 통제할 수 있는 방식입니다.
2. **모듈형 씬 구조**: 한 씬에 모든 것을 넣지 않고, 기능별로 씬(스크립트 단위) 작게 쪼개어 AI가 부분적으로 파일을 수정할 수 있는 안전한 환경을 유지합니다.
